AICO is a voice-agent runtime. It orchestrates conversations over WebRTC,
SIP, or WhatsApp through a DAG-based flow executor that calls swappable
STT, TTS, and LLM providers.

## Stack

| Layer | Implementation |
|---|---|
| Frontend | Svelte 5 · Vite · XYFlow (flow builder) |
| Backend | Bun · TypeScript · Elysia · Drizzle ORM |
| Database | PostgreSQL 17 + pgvector (managed by CloudNativePG) |
| Cache | Valkey 8 |
| Media | LiveKit 1.9.8 (WebRTC + SIP) · LiveKit Egress (recording) |
| Agent worker | Python · LiveKit Agents SDK · Silero VAD |
| STT service | FastAPI, Python, runs local STT engines |
| TTS service | FastAPI, Python, runs local TTS engines |
| LLM (local) | Ollama (AMD ROCm or NVIDIA CUDA) |
| Auth | Logto (OAuth2 / OIDC) |
| Embeddings | Ollama embeddinggemma (768-dim) |

## Services on the lab k3s cluster

| Namespace | Workload |
|---|---|
| `aico` | `backend`, `frontend`, `agent-worker`, `stt-router` + per-provider workers (`stt-vosk`, `stt-whisper`, …), `tts-router` + per-provider workers (`tts-piper`, `tts-kokoro`, …), `ollama` (StatefulSet) |
| `aico-data` | `aico-db` (CNPG Postgres), `valkey` |
| `aico-media` | `livekit`, `livekit-sip`, `livekit-egress`, STUNner UDP Gateway |
| `aico-auth` | AICO-branded `logto` + `logto-db` (CNPG) |
| `aico-system` | Cilium Gateway, cert-manager, shared ClusterSecretStore |

Per-provider STT / TTS workers are pinned to one engine each (`STT_PROVIDER_NAME`,
`TTS_PROVIDER_NAME`); the router dispatches by name. Hardware variant
(`cpu` / `rocm` / `cuda`) is picked per provider per cluster — ROCm
pods need `/dev/kfd` + `HSA_OVERRIDE_GFX_VERSION=11.0.0` on RDNA3, so
they're node-selected onto workers with that label. The full matrix
lives at `infrastructure/cdk8s/src/lib/inference-matrix.ts`.
